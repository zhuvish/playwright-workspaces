# Project Config Conventions

How to extend a Playwright project's configuration without modifying the user's source of truth.

## Rule: never modify the project's `playwright.config.ts`

The project's own config is the user's source of truth. Modifying it silently changes how `npx playwright test` behaves for everyone on the team. Don't do that.

Use **sibling configs** that extend the base and are passed via `--config=...` only when needed.

The **one** exception: adding `import 'dotenv/config';` at the top when tests need env vars and it's not already there (a strictly additive change that breaks nothing). Add `dotenv` to `devDependencies` in the same edit.

## Sibling config patterns

### `playwright.service.config.ts` ŌĆö for Azure Playwright Workspaces (cloud browsers, REQUIRED)

`/verify` is cloud-only. This file is generated by the skill's Step 0 bootstrap (see [cloud-setup.md](./cloud-setup.md)) and routes `npx playwright test` through the user's Azure Playwright Workspace. It is committable ŌĆö it references the workspace by resource ID only and contains no secrets.

```ts
import { defineConfig } from '@playwright/test';
import { createAzurePlaywrightConfig } from '@azure/playwright';
import { DefaultAzureCredential } from '@azure/identity';
import baseConfig from './playwright.config';

export default defineConfig(
  baseConfig,
  createAzurePlaywrightConfig(baseConfig, {
    credential: new DefaultAzureCredential(),
    connectTimeout: 3 * 60 * 1000,
  }),
  { workers: 10 },
);
```

Requires `npm install --save-dev @azure/playwright@1.1.5 @azure/identity@4.13.1`.

### `playwright.video.config.ts` ŌĆö for video recording (Final Sanity)

Extends the **service config** (not the base config) so videos are recorded against cloud browsers.

```ts
import baseServiceConfig from './playwright.service.config';
import { defineConfig } from '@playwright/test';

export default defineConfig(baseServiceConfig, {
  use: { video: 'on' },
});
```

Run with: `npx playwright test --config=playwright.video.config.ts`

## Reporter overrides ŌĆö use CLI flags, not config

For diagnostic runs (healing, verification) you usually want structured output:

```bash
npx playwright test --config=playwright.service.config.ts --reporter=json
```

**Never** modify the project's `playwright.config.ts` reporter setting. Always use the `--reporter` CLI override.

## What goes where, summary

| Need | Where it lives |
|---|---|
| Real-user runtime config | `playwright.config.ts` (project's own ŌĆö don't modify) |
| Loading `.env` | `playwright.config.ts` (one-time additive change is OK) |
| `.gitignore` for `.env` / `.env.local` | See [credential-handling.md](./credential-handling.md) |
| Cloud-browser execution (required) | `playwright.service.config.ts` sibling, used via `--config=...` |
| Video recording (Final Sanity) | `playwright.video.config.ts` sibling extending the service config |
| Reporter for diagnostic runs | CLI flag `--reporter=json` |
| Per-run timeout / retries | CLI flag `--timeout` / `--retries` |
